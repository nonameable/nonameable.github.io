Prodicy

Since I started this blog is been a pain in the ass writing the entries. Either I listen to the classes or watch the lectures, or I write good, or better blog post. Why I take too long? Maybe because I don't type fast enough. Maybe I don't have a good system of note taking. On the other hand, I can easily describe what I've just learned while talking to someone. Why not talk to the computer? Wy not just find a way to simply speak, and be able to write a blog post, and if I want, add HTML to it so I can make it appropiate for a website?

I started searching and found Dictate 4.0, a software whuch apparently is ok; the problem is, it costs 199 dolars and I don't have that money. That reminded me of the dictation feature in OSX , but as soon I tried to use it, I realize that the computer was going to send the audio to Apple in order to translate it and then return it back to me.
Also, I didn't quite liked tha tthe computer could only listen for up to 30 seconds at a time, which is not that cool. I want something that listen continously. I need to build something myself.

Inspiration and code

This is the first time I decide to develope something by myself. I know it's going to be hard. I need inspiration, other projects to support my own; here are a few I found that maybe could help me build better software:

  • Palaver (forgotten Ubuntu Siri-like recognition software.)

    Apparently this college studnet James mcClain use Google's voice API and database on the backend to make this program. Here is the repo Palaver

  • Natl , muti-language voice control system.

    This was made in a similar way to Palaver, but seems to be just for the computer, not or Internet searches.

  • Dictation Toolbkox

    This guy wanted to code by voice: using python to Code by Voice

The first thing I need is speech recognizition software. As I must remember every time I start a new project: I must stand upon the shoulders of giants. I started searching and found the following libraries:

  • pygsr
  • SpeechRecognitiion 2.0.1
  • Web speech API from Google

    iF the API is not opensource, could I make a web app do the trascript and then present the results to the user? Check this out: HTML5 speech recognition API

  • J.A.R.V.I.S

    • Apparently, Google shutdown the V1 API. Luckily, there is handy unofficial Java implementation for the new v2 version located in the link above. A key is required to get it working. You need to also subscribe to the chromium developers newsletter to be able to see the option for that key. Full instructions are included on the wiki of the project. I think I'm gonna go with the Java library, as I think learning python to the point where I feel confortable using the API.
