Whenever numbers get too big weird things start to happen. Why? In this class we will consider the anomalies with integers. It turns out, each integer type in Java (and in other languages, of course), has a range of possible values.

How How are we supposed to handle Overflow? Simply, the minimum number is one less than the maximum number.

Minimums and Maximums and it's relation to Java

The image above shows the Minimums and maximums table for both US and S.

The key thing to take into account is that in both US and S arithmetic once you get to the biggest number, it returns to the somallest one by truncation and in the case of S, also by Twos complement. This is the reason for some of the weird things happening in Java with big numbers, in this case, integers
Oh, so you don't believe me, try this:
System.out.println(2147483647); Result: 2147483647.
System.out.println(2147483647 + 1); Result: - 2147483648.
This works with bytes too!
System.out.println( (byte) (127) ); Result: 127.
System.out.println((byte) (127+ 1) ; Result: - 128.
